RAMS · Pallet · Inventory Reconciliation

Field Audit — Inspector App

Mobile & tablet prototype for the auditor, styled with the exact tokens from rams-irds-main (theme.css / tokens.ts) — same colors, radius, and type scale as the web app, just resized for a handheld screen.

Log in → Resume/Start an audit → Scan a pallet, count SKUs, Save Record → check Progress. Every KPI updates from real state. On Tablet, the left rail gives direct navigation.

Data model — what backs this app

Already live in Supabase — Dashboard, My Audit Tasks and Audit Details read directly from these:

inventory_reconciliation_audit_plans
  audit_id, audit_name, audit_type, count_method,
  scope_type, scope_values, scope_config, team_members,
  start_date, end_date, status, progress

inventory_reconciliation_records        -- the blind "expected" side
  inventory_id, sku, sku_name, pallet, qty,
  zone, location, lot, status

New — proposed the Audit Count Sheet and Audit Progress screens need two tables that don't exist yet:

inventory_reconciliation_locations      -- rack → bay → storage location
  id, audit_id, rack_code, bay_code, location_code,
  parent_bay_id, status  ('Not Started'|'In Progress'|'Completed')

inventory_reconciliation_count_records  -- what the inspector scans
  id, audit_id, location_id, pallet_id, sku, lot,
  counted_qty, condition, scanned_by, scanned_at

-- Rack/Bay/Location "Completed" status and every KPI in this prototype
-- are computed live from these two tables, not stored as a manual number.
-- Scope-locked fields on Audit Details (Rack/Bay fixed to "Only option")
-- are derived the same way: fixed when a level has exactly one row in
-- scope, a dropdown when it has more than one.
